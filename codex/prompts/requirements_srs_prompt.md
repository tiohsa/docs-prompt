# 出力要件

1. 出力は **日本語の Markdown** とし、テンプレートの章立て・見出し構造を維持してください。
2. テンプレート内の「説明文」や「TBD」などのプレースホルダを、可能な範囲で
   `{{project_info_yaml}}` の内容から具体的な文章・表・箇条書きに置き換えてください。
3. 情報が不足している場合は、無理に推測せず、以下のいずれかの方法で明示してください。

   * 「前提条件」セクションに、仮置きの前提を書く
   * 「未確定事項」リストに、決まっていない内容を書く
   * 「要確認事項・質問リスト」に、関係者に確認すべき具体的な質問を書く
4. 少なくとも以下の観点は章内のどこかで必ず触れてください。

   * 背景・目的
   * 対象範囲（スコープ）と非スコープ
   * システム利用者・ステークホルダー
   * 機能要件（概要レベルでよい）
   * 非機能要件（性能・可用性・セキュリティ・運用など）
   * 制約条件（技術・組織・スケジュール・予算など）
   * 想定リスクと対策の方向性
5. 表や Mermaid がテンプレートに含まれている場合：

   * 可能な部分は具体的な値・項目名で埋めてください。
   * 情報が不足しているセルには `TBD` と記載してください。
